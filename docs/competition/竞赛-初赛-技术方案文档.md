# ScholarFlow 端侧 AI 学习助手 —— 初赛技术方案
### 赛事：手机上的创意AI ｜ 参赛手机：iPhone 15 ｜ 模型：Qwen3-0.6B ｜ 框架：MNN-LLM

> **一句话亮点：把大学生分散在十几个 App 里的学业事务聚合进一个 App，再用一个随身、离线的端侧 AI 助手统一驱动——极致方便。**

---

## 应用场景

大学生的学业事务被打散在十几个 App、网页和工具里——查课表一个、记作业一个、算 GPA 一个、记笔记一个、抢图书馆座位又一个，来回切换、还各自要登录联网。

**ScholarFlow 把这些全部聚合进一个 App**（课表、作业、考试、GPA、知识库、番茄钟、图书馆选座…）。本项目要做的，是让一个**完全跑在手机本地的 AI 助手**成为贯穿所有模块的**统一智能入口**：

- **图书馆 / 自习室**：随手问一道高数 / 编程 / 统计题，离线即答
- **复习周**：让它按你聚合在这里的知识库 / 作业出练习、规划复习时间表
- **任何时候**：一个 App、一个随身离线助手搞定全部学业——不用在多工具间反复切换、不联网、不付费

## 用户痛点

| 痛点 | 现状 | 本项目方案 |
|---|---|---|
| **工具分散** | 课表 / 作业 / 笔记 / 选座散落在多个 App 与网页，来回切换 | **聚合到一个 App**，一个 AI 统一入口 |
| 网络依赖 | 图书馆地下层 / 宿舍 wifi 不稳、流量贵，断网即不可用 | **离线可用**，零流量 |

| 缺个人上下文 | 通用云 AI 不懂"我的课表 / 作业 / 知识库" | 与本地学业数据同处一端，可端侧个性化 |

## 技术方案（模型选型 + 推理框架 + 端侧适配思路）

### 模型选型：Qwen3-0.6B（4-bit 量化）
- 中文与学科推理能力强，开源、可端侧部署
- 0.6B 主打「极小体积 + 快加载 + 低内存」，在 iPhone 15 本地 CPU 上加载快、推理稳；比 1.7B 更轻（1.7B 可作更高质量备选）
- MNN 官方已提供预转换权重 `MNN/Qwen3-0.6B-MNN`（魔搭 ModelScope），4-bit 约 0.4GB，**免自行转换**

### 推理框架：MNN-LLM（alibaba/MNN）
- 端侧专用推理引擎，CPU 上 prefill / decode 较 llama.cpp 约 **8.6× / 8.9×**（MNN-LLM 论文数据）
- 官方提供 **iOS 端侧 LLM App** 作集成参考；与 Qwen 同源，适配成熟
- 支持 4-bit 量化、low-memory 模式、DRAM-Flash 混合存储，贴合移动端内存约束

### 端侧适配思路
- **模型分发**：不打包进 IPA，首启从 ModelScope 下载至 App 沙盒、再本地加载（控包体；此过程即"模型本地加载"演示点）
- **运行时**：MNN 编译为 iOS xcframework（`MNN_BUILD_LLM / LOW_MEMORY / 权重反量化 / Transformer 融合`），CPU(NEON) 后端
- **桥接**：Capacitor 原生插件（Swift → Obj-C++ → MNN C++），逐 token 流式回调，推理跑原生线程、不阻塞界面
- **聚合入口架构**：前端 `useChat` 抽象出 `ChatBackend`——手机走原生 MNN、桌面/开发走 Ollama，**同一套聊天 UI 零改动**；AI 助手作为统一入口，未来可端侧读取聚合在本地的学业数据（端侧 RAG）
- **内存治理**：针对 iPhone 15 内存预算做 4-bit + low-memory，0.6B 体积小、内存占用低（必要时可上探 1.7B）
- **SME2 前瞻**：iPhone 15（A16/A17 Pro）走通用 NEON；在含 SME2 的 **M5 上实测验证 MNN 的 SME2 加速路径**——形成"今天 iPhone 15 可跑、未来 A19 / 天玑9500 无缝吃 SME2 提速"的同构演进路线

## 创新点

1. **「聚合平台 × 端侧 AI」带来的极致方便（最大亮点）**：ScholarFlow 把大学生分散的学业全流程聚合到一端，端侧 AI 助手作为统一入口贯穿所有模块——一个 App、一个离线助手解决全部学业，免切换、免联网、免付费。这种"聚合 + 随身 + 端侧"的便利，是单点工具和云端 AI 都给不了的
2. **离线 / 零成本 / 隐私三位一体**：端侧推理让这个聚合平台随身可用、零流量、零付费、数据不出端
3. **AI 懂你的全部学业上下文**：数据都聚合在本地一端，AI 可端侧读取课表 / 作业 / 知识库做个性化辅导、零上传（端侧 RAG 方向）——聚合平台独有、单点工具给不了的能力
4. **可移植工程**：桌面 Ollama / 手机 MNN 双后台、UI 零改动的抽象；M5(SME2) → iPhone 15(NEON) 同构演进
