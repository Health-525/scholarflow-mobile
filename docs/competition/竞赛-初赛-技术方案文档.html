这套学习系统已经把"方便"做实——下列均为真实在用功能。本项目的核心升级，是把其中的 AI 大脑从云端搬进手机端侧、离线运行。

  • ✓AI 信息流：每日 AI 新闻、视频字幕自动总结。

技术方案

模型选型 + 推理框架 + 端侧适配，全部跑在手机本地

不接任何云 API。模型与推理都在 iPhone 15 上完成，桌面/开发态自动切回 Ollama，便于在 M5 上快速迭代。

项
选择
关键理由
模型选型
Qwen2.5-3B-Instruct · 4-bit
中文与学科推理强；3B 是手机端"质量 / 体积 / 速度"最佳平衡；阿里官方已预转换 MNN 格式，免自行转换（备 1.5B 应对低内存）。
推理框架
MNN-LLM
端侧专用引擎；CPU 上 prefill / decode 较 llama.cpp 约 8.6× / 8.9×（MNN-LLM 论文）；官方有 iOS 端侧 LLM App 可参考。
部署
iPhone 15 · NEON CPU
参赛机；MNN 编译为 iOS xcframework，CPU 后端运行，不依赖云、不依赖 NPU。
模型分发
ModelScope 直链 · ~1.9GB
不打包进 IPA，首次启动下载到 App 沙盒、再本地加载——这一步即演示视频要录的"模型本地加载过程"。

端侧架构：聊天 UI 零改动，手机走 MNN、桌面走 Ollama

1聊天 UI现有 useChat 界面，端侧 / 桌面共用一套，零改动。
2ChatBackend 抽象按平台分流：手机→原生，桌面/开发→Ollama。
3Capacitor 插件Swift → Obj-C++ 桥接，逐 token 流式回调。
4MNN-LLM 运行时原生线程推理，不阻塞界面。
5Qwen2.5-3B4-bit 权重，端侧本地生成。

端侧适配思路

1下载即加载首启从 ModelScope 拉取到沙盒、MNN 加载，控包体、可断点续传。
2xcframework 构建开启 LLM / low-memory / 权重反量化 / Transformer 融合等编译选项。
3流式不卡 UI推理跑原生线程，逐 token 事件推回 WebView 渲染。
4聚合入口架构AI 作统一入口，未来端侧读取本地学业数据（端侧 RAG）。
5内存治理4-bit + low-memory 贴合 iPhone 15 内存预算，3B 偏紧时降级 1.5B。
6无云、无 NPU纯 CPU 端侧推理，满足"本地加载 + 真交互"赛事要求。

SME2 前瞻：今天可跑，未来无缝提速

iPhone 15（A16 / A17 Pro）走通用 NEON CPU；我们在含 SME2 的 M5 上实测验证 MNN 的 SME2 加速路径，形成同构演进路线。

iPhone 15 · NEON → M5 · SME2 实测验证 → 未来 A19 / 天玑9500 无缝吃 SME2

注：A19/M5 硬件具备 SME2，但 MNN 在 iOS 上是否实际派发 SME2 kernel 以真机实测为准，不预先把加速倍数写死。

创新点

差异化不是"更聪明"，而是更方便——聚合 + 随身 + 端侧

它不是为比赛临时搭的 demo：源于一套真实在用的自动化学习系统，本项目把它的 AI 大脑从云端搬进手机本地、离线运行。最大亮点——方便。

01

「聚合平台 × 端侧 AI」的极致方便（最大亮点）

把分散在十几个 App 的学业聚合到一端，端侧 AI 作为统一入口贯穿所有模块——一个手机、一个离线助手解决全部学业，免切换、免联网、免付费。

02

真实在用，不是临时 demo

从"查课表"一路长成学习工作流系统（课表 / 作业 / 笔记 / 多端同步），作者每天在用——真实、可信，踩中 OpenClaw 端侧 AI 热点。

03

从云端自动化 → 端侧 AI

原本 AI 跑在云端（自动化脚本 / API）；本项目把核心 AI 落到手机本地（Qwen2.5-3B + MNN），离线可用、零成本、数据不出端。

04

AI 懂你的全部学业上下文

数据都聚合在本地，端侧读取课表 / 作业 / 知识库做个性化辅导、零上传（端侧 RAG 方向）。

05

可移植工程

桌面 / 手机双后台、UI 零改动的抽象；M5(SME2) → iPhone 15(NEON) 同构演进。
